# 使用 mcp-chrome 操作本机 Chrome

## 目标

通过 `mcp-chrome` 连接用户本机已打开的 Chrome / Chromium，会话中直接完成以下任务：

- 读取当前页面正文、HTML、标题与可交互元素
- 点击、输入、切换标签页、跳转页面、执行页面内脚本
- 截图关键区域或整页
- 抓取网络请求、响应体、控制台日志，辅助排障
- 复用用户现有登录态处理必须登录后才能访问的页面

## 何时调用

当用户提出以下诉求时调用：

- “用我现在打开的 Chrome 看一下这个页面”
- “复用浏览器登录态去操作网站”
- “帮我在网页里点按钮 / 填表 / 截图”
- “抓一下这个页面的请求 / 响应 / 控制台报错”
- “对当前打开的多个标签页做检索 / 汇总”

## 前置条件

只有在 `mcp-chrome` 已正确接入客户端时才直接执行浏览器操作；若尚未接入，先指导用户完成接入。

推荐接入方式是 `streamableHttp`：

```json
{
  "mcpServers": {
    "chrome-mcp-server": {
      "type": "streamableHttp",
      "url": "http://127.0.0.1:12306/mcp"
    }
  }
}
```

最小接入步骤：

1. 全局安装 bridge：`npm install -g mcp-chrome-bridge`
2. 打开 `chrome://extensions/`，开启开发者模式
3. 从 `https://github.com/hangwin/mcp-chrome/releases` 下载并加载解压后的扩展
4. 点击扩展图标并执行 connect
5. 在客户端中配置 `streamableHttp` 指向 `http://127.0.0.1:12306/mcp`

若客户端只支持 `stdio`，可退回：

```json
{
  "mcpServers": {
    "chrome-mcp-stdio": {
      "command": "npx",
      "args": ["node", "/path/to/mcp-server-stdio.js"]
    }
  }
}
```

其中 `mcp-server-stdio.js` 位于全局安装包的 `dist/mcp/mcp-server-stdio.js`。

## 入参建议

- `url`（可选）：目标网页；未提供时优先操作当前活动标签页
- `goal`（必填）：本次目标，例如“提取正文”“完成表单提交前检查”“抓取失败接口”
- `tab_hint`（可选）：标签页标题、URL 关键字或标签索引
- `need_screenshot`（可选）：是否输出截图
- `need_network_trace`（可选）：是否抓取网络请求与响应
- `need_console_logs`（可选）：是否读取控制台日志

## 工作流

### 1）确认浏览器会话

- 优先使用当前已连接的 Chrome 会话，不额外启动隔离浏览器。
- 如果用户给了 `url`，先判断是打开新标签还是在当前标签导航；没有明确要求时，优先复用当前标签。
- 如果有多个标签页，先根据 `tab_hint`、标题或 URL 关键字定位，再执行动作。

### 2）读取页面与定位元素

- 先读取当前页面 URL、标题、主要正文、可交互元素清单。
- 对读取类任务，优先提取正文 / HTML / 关键元素文本，不做多余点击。
- 对操作类任务，先识别按钮、输入框、下拉框、链接等候选元素，再执行最小必要操作。

### 3）页面交互

- 支持点击、输入、选择、快捷键、切换标签页、前进后退、刷新。
- 支持在页面中执行脚本读取运行时状态。
- 若任务涉及登录后页面，直接复用浏览器现有登录态，不要求用户重复输入凭证。

### 4）调试与排障

- 当用户要查接口、报错或页面异常时，开启网络与控制台采集。
- 优先输出：失败请求 URL、方法、状态码、请求参数、关键响应体、控制台错误。
- 若需要复现问题，按用户描述执行最短操作路径，并记录对应请求与报错。

### 5）截图与结果整理

- 用户要求“看页面效果”或“保留证据”时，再做整页或局部截图。
- 最终输出按任务类型整理为：结论、关键页面信息、关键元素、网络 / 控制台异常、截图路径。

## 执行原则

- 优先复用用户现有 Chrome 会话与登录态，不擅自打开无关页面。
- 优先读取，后交互；优先最小变更，避免多余点击。
- 未经用户明确要求，不执行有副作用的最终提交动作，例如“发布”“下单”“发送”“删除”“付款”。
- 如必须进入高风险动作前一步，停在确认页并向用户汇报当前状态。
- 不输出或持久化页面中的 token、cookie、验证码、个人隐私等敏感信息。

## 典型场景

- 读取当前网页并总结要点
- 复用登录态打开后台页面并检查配置
- 在网页中完成搜索、筛选、表单填写与截图
- 抓取前端报错、失败接口和关键响应体
- 在多个已打开标签页之间做内容检索与汇总

## 示例触发语句

- “用 Chrome 当前打开的页面帮我提取正文并截图”
- “复用我浏览器里的登录态，看看这个后台接口为什么 500”
- “帮我在网页里把表单填好，但先不要点提交”
- “抓一下这个页面加载时的请求和控制台报错”
